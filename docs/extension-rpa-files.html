一、总览与加载关系

扩展只做三件事：在已登录的美团/抖音页读 DOM、经本机 WebSocket 上报消息、收到草稿后回填或点发送。不登录平台、不读 Cookie/密码。大脑（RAG/LLM）都在本地 API。

manifest.json声明权限与脚本
→
background.jsService Worker
→
platform-profiles
+ content.js注入页面/iframe
→
本地 Gateway:3001/ws
→
draft → applyDraft回填 / 发送

隔离世界 Content

platform-profiles.js + content.js 在扩展隔离世界跑，读 DOM、点左侧会话、回填输入框。

MAIN world 辅助

抖音 React 有时忽略隔离世界点击；douyin-main-click.js 由 background 注入页面主世界补点。

弹窗配置

popup.* 读写 chrome.storage.local：选择器、监听/自动切会话/自动发送开关。

打开方式：用浏览器直接打开本文件；或从 extension-rpa-flow.html 跳转。行号针对当前仓库源码，大改后以函数名为准。

三、manifest.json

MV3 入口声明 改版本号后需 chrome://extensions 重载

扩展身份证：权限、可注入域名、脚本入口。改 host_permissions / content_scripts.matches 后必须重载扩展。

字段 / 区域作用
name / version / description扩展显示名与版本（当前以文件内 version 为准，如 0.3.30）
permissionsstorage 存配置；tabs/scripting/webNavigation 注入与路由；alarms 断线重连
host_permissions本机 API 127.0.0.1:3001、Mock :3100、美团经营宝、抖音来客
background.service_worker指向 background.js
content_scripts按顺序注入 platform-profiles.js → content.js；all_frames: true 覆盖 iframe
action.default_popup指向 popup.html

6.6 主循环 scanMessages（约 1426–1640）

  1. 遍历可访问 document → resolveSettings
  2. 上报 diagnostics
  3. scheduleUnreadConversation（自动切未读）
  4. readSession；白名单 / 系统会话 / chatReady 守卫
  5. maybeRecoverDraftsOnRefocus（找回 pending）
  6. 扫 inbound / outbound：初扫只记 seen；未读窗口末尾 N 条入队；抖音服务中客户可全量扫防连发丢句
  7. 系统提示只记 seen；同正文指纹去重；发 inbound/outbound 给 background

九、README.md

十、消息类型一览（扩展内部 + WebSocket）

content → background

  • session — 当前聊天身份
  • inbound / outbound — 客户/商家气泡
  • diagnostics — 诊断
  • request_drafts — 找回 pending
  • draft_send_result — 发送结果
  • clearFrameSession — 清空 frame
  • douyinMainClick — 请求 MAIN 点击

background → content

  • applyDraft — 下发回复草稿
  • rescan — 强制再扫
  • collectDomSnapshot — AI 识别快照
  • validateAiSelectors — 校验候选选择器

background ↔ 服务端协议由 apps/api/src/rpa/extension-gateway.ts 定义（hello、settings、inbound、draft、request_drafts 等）。逻辑流见 extension-rpa-flow.html。
