一、项目定位

统一接入企业微信、美团到店和抖音来客消息，通过知识库检索、OpenClaw 话术生成和风险控制，输出自动回复或人工审核草稿。平台接入必须经过 Adapter，不允许 OpenClaw 直接操作平台。

官方接口

企业微信

使用官方 Webhook/API 接收和发送消息，默认关闭自动发送。

RPA Adapter

美团到店

Chrome 扩展监听已登录页面 DOM，通过本地 WebSocket 与 API 通信。

RPA Adapter

抖音来客

当前保留 RPA 适配边界，不编造未确认的官方客服接口。

五、Hybrid RAG

Query Rewrite

将客户口语改写为多个检索问法，支持停车、预约、营业时间等表达差异。

Intent Classifier

识别 price、refund、reservation、parking、package 等意图，严格问题使用更高阈值。

Hybrid Retrieval

综合向量、中文关键词、平台/门店/分类 Metadata 和 Graph 关系。

Rerank

可选使用 LLM 重排；失败时回退 Hybrid 分数，不能阻塞客服主链路。

阈值

普通问题低于 0.68、严格问题低于 0.75 时转人工。

Gap Detector

未命中、低置信度和人工接管问题写入知识缺口，供后续补充卡片。

六、Chrome 插件多会话 RPA

  1. 插件通过 ws://127.0.0.1:3001/rpa/extension/ws 连接 API。
  2. 监听左侧未读会话并串行点击切换客户，默认兼容 .chat-list-item-wrapper、.chat-list-item 和 .virtual-list-item。
  3. 美团经营宝未读数字按 .mtd-badge-text.mtd-badge-position 判断，例如角标文本 1；避免使用过宽的 .mtd-badge 容器误判。
  4. 读取真实 shopId + conversationId + messageId。
  5. 线上灰度可通过 MEITUAN_RPA_ALLOWED_CUSTOMERS 限定只处理指定测试客户；入站、出站、会话注册和草稿查询都会服务端兜底校验。
  6. 当前页面若停在非白名单客户，插件会清理该 frame 的旧会话映射并释放调度锁，继续寻找白名单未读客户。
  7. 只用该会话最新一条未读消息触发回复，避免连续拆句生成多份回答。
  8. 最多等待 120 秒，草稿返回后再次校验当前门店和客户。
  9. 低风险草稿若暂未满足自动点击条件，会先回填输入框；美团白名单灰度会话在 low 风险、开关已开启、连接后新生成且通过冷却时点击页面发送按钮。
  10. 页面观察到 outbound 后，将对应 ReplyDraft 标记为 sent。
Mock 自动消息使用 30–70 秒随机空档；最近 10 条问题不重复，且不会继续向已有未读积压的客户生成消息。Mock 能力不能等同于真实平台生产稳定性。

七、上下文、去重与草稿关联

会话隔离键

platform + shopId + conversationId + customerId。不能使用昵称或页面标题作为生产会话标识。

消息与草稿

平台 messageId 做第一层去重，数据库做第二层去重。自动模式只接收当前浏览器实际提交 messageId 对应的草稿，历史 pending 不得重放。

历史上下文

近期消息直接提供给 OpenClaw，较早消息写入数据库摘要；订单实时状态不进入长期摘要。

发送确认

只有商家消息真实出现在页面 DOM 后才记录 outbound，并关闭相同会话、相同内容的待发送草稿。

订单路由优先于知识库：当前句包含查单意图，或最近客服正在等待订单号/手机号时，纯订单号会直接进入 Order Adapter，完全跳过 Hybrid RAG。只有非订单消息才执行知识库检索。

接口超时、鉴权失败、缺少订单号或查不到数据时必须转人工。未知企业接口继续保留 Adapter/TODO，不得编造官方接口。

十二、日常操作顺序

  1. 执行 pnpm dev 并确认 Docker、OpenClaw、3001、3100、8787 正常。
  2. 打开 http://127.0.0.1:8787/kb-admin 上传并解析文件。
  3. 点击文件“编译”，审核 Wiki 和 Knowledge Cards。
  4. 按需设置卡片平台、门店、分类、关键词和优先级。
  5. 在 chrome://extensions 重新加载修改后的插件。
  6. 开启页面监听和自动处理未读；确认安全后再单独开启自动发送。

开发启动

pnpm dev 会启动 API、RAG、Mock 页面，并按扩展模式等待 Chrome 插件连接。只有设置 RPA_MOCK_MODE=playwright 时才使用旧 Playwright watcher。

Windows 便携交付

pnpm package:windows 会生成一体化目录包，包含生产构建、干净 OpenClaw、Chrome 扩展、文档和启停脚本。客户双击 Start-Customer-AI.bat 后自动启动 Docker、OpenClaw、RAG 和 API。

首次客户配置

客服电脑仍需安装 Docker Desktop 和 Chrome；首次在 OpenClaw 配置模型 Key，并在 chrome://extensions 加载 extensions\customer-ai-rpa。真实平台登录态由客户自己的 Chrome 保存。

交付安全边界

打包脚本不会复制本机 .env、OpenClaw data、Cookie、聊天记录或日志。自动发送默认关闭，需客户确认后在插件中单独开启。

十三、推荐排障顺序

没有生成回复

  1. 检查 3001 WebSocket 状态。
  2. 确认 inbound 是否入库。
  3. 检查 BullMQ waiting/active/failed。
  4. 确认 RAG 是否返回证据。
  5. 检查 OpenClaw 超时或错误。
  6. 查看 ReplyDraft 风险和状态。

答非所问或重复发送

  1. 核对 messageId 与 draft.messageId。
  2. 检查旧 pending 是否被重放。
  3. 确认当前 conversationId 未切换。
  4. 检查 outbound 是否把草稿标记 sent。
  5. 确认没有多个 Mock 页面同时监听。

十四、当前边界

美团和抖音仍属于页面 RPA，真实平台 DOM、虚拟列表、风控策略和人工操作竞争需要真实账号灰度验证。生产前还需完善持久化调度、监控告警、人工接管工作台和完整发送审计。企业微信、抖音、美团未确认的官方接口必须继续保留 Adapter/TODO。
