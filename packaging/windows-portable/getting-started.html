首次配置流程

  1. 1

    确认 Docker 已启动

    需本机 Docker Desktop。交付包若含 runtime\docker-images\*.tar，启动会先本地 load，不必连 Docker Hub。空库会自动建业务表；RAG 表随服务初始化。能打开本页通常表示库与服务已就绪——接着配置 Key、上传知识库、加载扩展即可使用。

  2. 2

    填写大模型与 Embedding

    在本页右侧「大模型 / Embedding」中选择 Agnes、千问（DashScope） 或 自定义 OpenAI 兼容接口，并配置向量模型。保存后立即生效，无需重启。

  3. 3

    加载 Chrome 扩展

    打开 chrome://extensions，开启开发者模式，加载本便携包目录：

    extensions\customer-ai-rpa

  4. 4

    登录美团经营宝并开启插件

    在 Chrome 中人工登录客服页面。点击插件图标，确认显示「已连接本地服务」，并勾选：

    • 启用页面监听
    • 自动处理未读会话
    • 允许自动点击发送（测试通过后开启）
  5. 5

    上传门店知识库

    打开知识库管理页，上传门店地址、套餐、退款规则等资料。AI 回复必须基于知识库检索结果。

  6. 6

    配置 RPA 客户白名单

    在右侧「客户白名单」中填写可自动回复的客户 ID/昵称。美团灰度建议先加测试客户；全部清空后表示所有客户都可以回复。抖音默认也可留空。

大模型（客服回复）

协议为 OpenAI 兼容 /chat/completions。选 Agnes / 千问会填入官方默认；其它厂商请选「自定义」。千问 Key 在 千问云文档 获取（环境变量名多为 DASHSCOPE_API_KEY）。

Embedding（知识库向量）

OpenAI 兼容 /embeddings。请保证模型输出维度与当前 EMBEDDING_DIM 一致，否则需重编译知识库。

客户白名单

每行一个客户 ID 或昵称，也可用英文逗号分隔。清空并保存后，对应平台对所有客户开放自动回复。

安全提示：便携包默认建议先在草稿模式验证。开启 RPA_AUTO_SEND_ENABLED=true 前，请确认知识库、白名单和发送按钮选择器均正确。 不要删除 data 目录，否则登录态会丢失；也不要把客户使用后的整个目录对外转发。
